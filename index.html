How Chess can help you

Chess is a framework for creating single page javascript applications. This is NOT MVC framework. All idea behind Chess is declarative layout of reusable components. Chess is written in typescript and depends on Require.js and Zepto.js 

  
    var board = {
          'Title':'This is a title',
          'Body':{
             'LeftColumn':{
             }
             'RightColumn':{
                 'Button':'Click me'
             }
          'Footer':null    
    }

Nevermind how complex your app is, traversing html elements in firebug or webkit dev tools will be very easy, because html classes generated will match exactly with appropriate declarations: 

  
    <div class="Title">'This is a title</div>
    <div class="Body">
      <div class="LeftColumn"></div>
      <div class="RightColumn">
           <div class="Button">Click me</div>
      </div>
    </div>
    <div class="Footer"></div>

Nevermind how complex your app is, changing styles for such an app is very easy. Especially when yo use SASS, cause your stylesheet rules could (Chess does not yet generate css) match exactly with appropriate declarations:

  
    .Title{
        font-size:14px;
    }
    .Body{
        .LeftColumn{
            foat:left;
        }        
	.RightColumn{
            foat:right;
        }
        &:after{
            clear:both;
        }
    }
    .Footer{
        margin-top:20px;
    }

And finally, you can, for sure, hook in any place of your application with your typescript/javascript code. 

  
    export class Button{
        afterAppend(){
            $(this.el).on('click', function(){
                  alert('It works')
            })
        }
    }

Declaration Rules

While declaring your components you can use next modificators: 

    
var board = {
    'EmptyEl':null,
    'TextInEl':'Just a text',
    'HtmlInEl':'<span>HTML</span>',

    'Unique#UID':null,
    'Row.First':null,
    'Row.Second.Last':null,

    'Container':{
        'Child':{
              'Level1':{
                  'Level2':null               
               }
         },
         '_Delayed':{
               'Level1':{
                   '_Delayed2'null
               }
         }
    }    
}
    
  
    

<div class="EmptyEl"></div>
<div class="TextInEl">Just a text</div>
<div class="HtmlInEl">HTML</div>

<div class="Unique" id="UID"></div>
<div class="Row First"></div>
<div class="Row Second Last"></div>

<div class="Container">
  <div class="Child">
    <div class="Level1">
      <div class="Level2"></div>
    </div>    
  </div>
</div>

Rather Drive Stick?

If you prefer to not use the automatic generator, push a branch named gh-pages to your repository to create a page manually. In addition to supporting regular HTML content, GitHub Pages support Jekyll, a simple, blog aware static site generator written by our own Tom Preston-Werner. Jekyll makes it easy to create site-wide headers and footers without having to copy them across every page. It also offers intelligent blog support and other advanced templating features.
