How Chess can help you

Chess is a framework for creating single page javascript applications. This is NOT MVC framework. All idea behind Chess is declarative layout of reusable components. Chess is written in typescript and depends on Require.js and Zepto.js 

  
    var board = {
	  'Title':'This is a title',
	  'Body':{
	     'LeftColumn':{
	     }
	     'RightColumn':{
		 'Button':'Click me'
	     }
	  'Footer':null
    }

Nevermind how complex your app is, traversing html elements in firebug or webkit dev tools will be very easy, because html classes generated will match exactly with appropriate declarations: 

  
    <div class="Title">'This is a title</div>
    <div class="Body">
      <div class="LeftColumn"></div>
      <div class="RightColumn">
	   <div class="Button">Click me</div>
      </div>
    </div>
    <div class="Footer"></div>

Nevermind how complex your app is, changing styles for such an app is very easy. Especially when yo use SASS, cause your stylesheet rules could (Chess does not yet generate css) match exactly with appropriate declarations:

  
    .Title{
	font-size:14px;
    }
    .Body{
	.LeftColumn{
	    foat:left;
	}
	.RightColumn{
	    foat:right;
	}
	&:after{
	    clear:both;
	}
    }
    .Footer{
	margin-top:20px;
    }

And finally, you can, for sure, hook in any place of your application with your typescript/javascript code. 

  
    export class Button{
	afterAppend(){
	    $(this.el).on('click', function(){
		  alert('It works')
	    })
	}
    }

Declaration Rules

While declaring your components you can use next modificators: 

    
var board = {
    'EmptyEl':null,
    'TextInEl':'Just a text',
    'HtmlInEl':'<span>HTML</span>',

    'Unique#UID':null,
    'Row.First':null,
    'Row.Second.Last':null,

    'Container':{
	'Child':{
	      'Level1':{
		  'Level2':null
	       }
	 },
	 '_Delayed':{
	       'Level1':{
		   '_Delayed2'null
	       }
	 }
    }
}

Application

When declare your components it is not required to instantiate them in your code. It is not even required to have all classes used at your board. Most time components are used for generating empty markup to style it via sass.

  • All components are based on BaseCell class in chess/piecess module.
  • Main component to which your application is attached for is called Viewport (pieces.Viewport class) You can have several application on page each attached to its viewport.
  • First level components in board (the board variable) are called Screens (pieces.Screen class) You can have several screens in your application (separate screens for mobile app, modal forms etc).
To start you application, you must instantiate it with the component declaration and function which will choose screen for initial render. Here is the hello world: 
    
      ///<reference path="jquery.d.ts"/>
      import chess = module("chess/app")
      import interfaces = module("chess/interfaces")
      import pieces = module("chess/pieces")
      
      var board = {
         'Screen':'Hello world'
      }

      var viewport = new pieces.ViewPort($('body')[0])
      var application = new chess.ChessApp(viewport,board, [])
      var selector = function(screens:interfaces.ScreenMap){
             var answer = screens['Screen']
             return answer
      }
      application.resolve(selector)
Third parametr in application constructor is a list of modules, from which application will select classes declared in board. If no class will found, Chess will take BaseCell for that board record.
